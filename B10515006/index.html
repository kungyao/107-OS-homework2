🤔 功能說明

在我們這次實作的system call中，完成了以下system call：

  • chown(const char*, int)
  • chgrp(const char*, int)
  • chmod(const char*, int)
  • getuid(void)
  • getgid(void)
  • su(int)

system call說明：

  • chown：切換檔案擁有者
  • chgrp：切換檔案擁有群組
  • chmod：切換檔案權限
  • getuid：獲取當前使用者ID
  • getgid：獲取當前使用者群組ID
  • su：切換當前系統使用者

透過上述新增的system calls，xv6得以擁有了類似於Linux的使用者群組的概念，並且在檔案中增加了擁有者、群組與存取權限等的資訊，讓檔案系統增加了安全性與保密性。

😇 實作過程(修改哪些檔案[含圖片])

主要修改的檔案：

Makefile

因為新增了一些user application來供shell使用，因此makefile需要增加相關參數在UPROGS內。
另外ADDITIONFILES是用來將額外的文件放進xv6所創的變數，目前有新增了一個NEWFILE檔案來測試。

ls.c

額外輸出uid / gid / mode的資訊。

proc.c

我們將當前user和group的資訊塞入process內儲存，系統剛建立時的process的uid / gid預設為0。
fork時會將uid與gid複製過去。

proc.h

stat.h

syscall.c

def.h

chkMode用來確認當前的操作是否合法，實作在 sysFile.c。

sysfile.c

int chkMode(short uid, short gid, uint mode, char rwx)

用於檢查當前使用者/群組是否可以對檔案進行指定操作(read/write/execute)。
我們透過建立一個9bit的mask與檔案的mode做AND運算，若結果大於0則return 1(接受)，否則return 0則(拒絕)。
ROOT的uid = 0，而ROOT擁有最高權限可以進行任何操作。

sys_read(void)、sys_write(void)

對sys_read與sys_write進行權限控管

create(char *path, short type, short major, short minor)

sys_chown(void)

用ilock鎖住inode後更改uid內容並iupdate。

sys_chgrp(void)

用ilock鎖住inode後更改gid內容並iupdate。

sys_chmod(void)

用ilock鎖住inode後更改mode內容並iupdate。

exec.c

執行權限的控管

sysproc.c

sys_getuid(void), sys_getgid(void)

直接回傳當前process的uid / gid

sys_su(void)

從當前的process找parent找到最上層，沿路更換它們的uid來達成切換使用者的功能。

user.h

usys.S

fs.h

這裡的NDIRECT從12修改為10很重要，因為在建立file system的時候會做一次size檢查，這裡的sturct因為我們增加了8byte的額外資訊，因此會在檢查中fail。
我們在這裡debug好一段時間，最後的解決方法就是把原本的addrs縮小了8bytes。

User applications

供shell來執行，以便測試我們的system calls功能。

getuid.c:

getgid.c, chown.c, chgrp.c, su.c

程式邏輯皆類似於getuid.c，皆是直接call system call後printf。

chmod.c

由於Linux的file mode顯示是用三組0~7的值分別對應owner/group/other的 read / write / execute 權限 (如700代表rwx------，770代表rwxrwx—)，但是我們mode實際上是直接以9 bit的int變數來儲存，因此輸入與輸出會額外經過一層轉換。

🐻 預期成效、情境說明（怎麼用、誰用）

仿造Linux系統實作出上述的system call，在xv6作業系統中，新增有關系統使用者的相關操作，未來所有使用者若想在xv6上實現多人作業系統也更為方便。

之後我們希望能實作出更加完整的使用者系統，包含登入登出、帳號密碼等等，再利用這次實作的system call，自動的在使用者創建檔案的當下就設定好所有的權限管理，爾後如果使用者欲修改該檔案的資源與權限，也能很方便的進行修改的動作，達成在各個使用者之間進行資源、權限的調度，完成xv6多人作業的功能。
