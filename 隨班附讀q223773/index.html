🤔 功能說明(xv6 Page Table)

brk() and sbrk()

  • 兩個 system call 皆可用來改變 program break 的位置，但使用上有些微的差異。

brk()

int brk(void *addr);
傳入一個 addr，為程序的資料區段之結束位置所在，將程序挪移到此 addr 的位置並實現分配與回收。

sbrk()

void *sbrk(intptr_t increment);
傳入一個偏移量 increment，並將程序的位置挪移 increment 的大小。當 increment 為正值時往後移動 increment 大小，同時返回移動前之位置，相當於分配內存；當 increment 為負值時往前移動 increment 大小，相當於釋放內存；當 increment 為 0 時，不移動位置只返回當前位置。

由於 xv6 已將這兩個 system call 做了一個合併的設計，我們在這次作業中會給這個 sys call 一個偏移量來移動其 address，並多設計一個 deallocate 的機制，此外，在作業三中會實作 lazy allocation page table 的機制。

cps

  • 打印目前所有 process 的狀態 (name, pid, size, and state)

calloc() and ckalloc()

calloc():

藉由這個system call 可以讓 user 在 terminal 直接呼叫方便 demo
calloc 後會接一個 size 大小，由 user 自行決定。
後面的 nbytes 已經先固定成一個 page 的大小 4096。
calloc() 會呼叫 calloc_ 這個 function 來執行要空間的動作。

ckalloc():

因為原本 malloc 的機制並不會要求連續空間，因此中間加了 cmalloc, csbrk, cgrowproc, callocuvm 等，讓它整個流程最後呼叫的是 ckalloc()
ckalloc 會進行判斷，確認下個空間是否為連續的，若可以才回傳地址（相鄰的位置互差 1024）

malloc() and realloc()

malloc() vs calloc()

將空間分配於 heap，malloc 沒有初始化內容，calloc 則有。

malloc() vs realloc()

realloc 一樣將空間分配於 heap，將 malloc 出的空間進行大小縮放，如果沒有傳入的 header 存在則進行初始化。如果不足則分配一個新的位址將資料搬過去，原本的則 free 掉。

urealloc()

為了方便 從terminal demo 並理解用。

知識補充:

  • virtual address 中文線性位址，又稱虛擬地址。
  • intel 自從 32 bit cpu 出現(80386)，logic address = virtual address.
  • logic memory is a memory that combines every physical memory.
